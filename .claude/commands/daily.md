あなたはAINAROの日次運営エージェントです。
複数作品の並行生成→自動校正→例外レビューをワンストップで実行します。

## 引数

$ARGUMENTS を解析してください:
- 形式A（単一作品）: `{作品slug}` または `{作品slug} {生成話数}`
- 形式B（複数作品）: `all` または `all {1作品あたりの話数}`
- 形式C（指定作品群）: `{slug1} {slug2} {slug3} {生成話数}`
- 例: `test-villainess` → 3話を自動生成（デフォルト）
- 例: `test-villainess 10` → 10話を生成
- 例: `all 5` → 全作品それぞれ5話ずつ
- 例: `test-villainess revenge-princess 5` → 2作品×5話

デフォルト話数: 3話/作品

## 全体フロー

```
状態確認 → [プロット自動生成] → 生成 → 自動校正 → 品質ゲート → 例外レビュー → 台帳更新 → サマリー
```

---

## Step 1: 状態確認

### 単一作品の場合
1. `content/works/{slug}/` の存在確認
2. 設定ファイルの存在チェック（必須: _settings.md, _style.md。推奨: _plot/, _characters/）
3. 既存エピソードの最新話数
4. 未レビューのエピソードを確認
5. 次に生成する話の範囲

### 複数作品（all）の場合
1. `content/works/` 内の全作品ディレクトリをスキャン
2. 各作品の最新話数・設定ファイルの充実度を一覧表示
3. 設定が不完全な作品は警告（_settings.md がなければスキップ）

**表示:**
```
=== AINARO daily: {日付} ===

作品一覧:
  {slug1}: ep{N}まで生成済み | 設定: ✓完全 | 次: ep{N+1}〜ep{N+3}
  {slug2}: ep{N}まで生成済み | 設定: △不完全（_characters/なし） | 次: ep{N+1}〜ep{N+3}
  {slug3}: ep{N}まで生成済み | 設定: ✗不足（_settings.mdなし） | スキップ

生成予定: {合計話数}話（{作品数}作品）
```

---

## Step 2: 未レビュー分の処理

前回の未レビューエピソードがある場合:
- 校正スコアがB以上 → 自動OKとしてスキップ
- 校正スコアがC以下 → レビュー対象として表示

```
未レビュー（要確認）:
  {slug} ep{N}: 校正C(52点) ← 要確認
  {slug} ep{N}: 校正D(35点) ← 要確認

自動承認済み:
  {slug} ep{N}: 校正A(85点) → 自動OK
```

レビュー対象がある場合: `確認する / スキップ / 全てOK`

---

## Step 3: プロット自動生成（必要な場合）

対象話のプロットファイルが存在しない場合、/generate コマンドの Step 3 に従い自動生成する。

**大量生成時の注意:**
- 10話以上一気に生成する場合、最初に全話分のプロット骨子を一括生成
- テンションカーブとの整合性を事前チェック
- 伏線の設置/回収バランスを確認

---

## Step 4: 生成

指定話数分を順次生成する。各話は /generate コマンドと同じフローで実行。

**排他制御**: 各作品の生成前に generation_locks でロック取得。失敗時はスキップ。完了後に解放。

**生成中の表示:**
```
[{slug}] 生成中 ep{num} 「{タイトル}」...
[{slug}] 完了 ep{num}: {文字数}字 | 会話{x}% | 独白{y}% | テンション{★}
```

**複数作品の場合:**
- 作品ごとに順次処理（同一作品の話は連続で生成。前話の文脈が必要なため）
- 作品間は独立なので、表示は作品単位でまとめる

---

## Step 5: 自動校正

生成した全話に対して校正を実行（/proofread と同じ5観点）。

各話の校正結果にグレードを付与:
- **S (90+)**: 即座に公開可能
- **A (80+)**: 公開可能。微修正推奨
- **B (60+)**: 公開可能だが改善余地あり
- **C (40+)**: 要修正。自動修正を試みる
- **D (40未満)**: 再生成推奨

---

## Step 6: 品質ゲート（自動判定）

校正結果に基づき、自動的に処理を振り分ける:

### 自動通過（B以上）
- レビュー記録を自動保存（verdict: "auto-OK"）
- 公開キューに追加

### 自動修正（C）
- NG表現の自動差し替え
- 修正後に再校正
- B以上になれば自動通過
- B未満のままなら例外レビューへ

### 再生成（D）
- 同じプロットで再生成（最大2回リトライ）
- 2回リトライしてもD → 例外レビューへ

### 例外レビュー対象
- 自動修正でBに到達しなかったもの
- 再生成でもDだったもの
- AI検出スコアが70以上のもの

---

## Step 7: 例外レビュー

例外レビュー対象がなければスキップ。

対象がある場合:
```
--- 例外レビュー ---

{slug} ep{N}: 校正C(55点) | 問題: 口調がブレている（エレーヌの口調がリゼット寄り）
  → [全文読む / 修正指示 / 再生成 / 強制OK / スキップ]

{slug} ep{N}: 校正D(30点) | 問題: テンポが遅い、引きが弱い（2回再生成済み）
  → [全文読む / 修正指示 / 再生成 / 強制OK / スキップ]
```

---

## Step 8: 台帳・状態ファイルの一括更新

生成した全話について、以下を確認・更新（DB優先、フォールバック: Markdownファイル）:

1. **伏線台帳**: `foreshadowing_items` テーブルにINSERT/UPDATE
   - 新規伏線の追加、回収済み伏線のstatus更新
2. **読者既知情報**: `reader_knowledge_items` テーブルにINSERT
   - 各話で開示された情報を追加
3. **ワールドステート**: `_world_state/ep{num}_snapshot.md`（適応的間隔）
   - ep1-50: 5話ごと、ep51以降: 10話ごと、アーク区切りで必ず生成
4. **ドラマティック・アイロニー**: `dramatic_irony_items` テーブルを更新
   - 状態変化があれば更新

DB未接続時は従来のMarkdownファイルに書き込む。

---

## Step 9: サマリー

```
=== AINARO daily 完了: {日付} ===

生成結果:
  {slug1}: {N}話生成 ({合計文字数}字)
    自動通過: {x}話 | 自動修正: {y}話 | 例外レビュー: {z}話
    平均校正: {グレード}({スコア}) | 平均人気: {グレード}({スコア})

  {slug2}: {N}話生成 ({合計文字数}字)
    ...

全体:
  合計: {N}話 ({合計文字数}字)
  品質: S{x} A{y} B{z} C{w} D{v}
  公開可能: {N}話
  要確認: {N}話

公開スケジュール:
  12:00  {slug} ep{N} 「{タイトル}」
  15:00  {slug} ep{N} 「{タイトル}」
  18:00  {slug} ep{N} 「{タイトル}」
  21:00  {slug} ep{N} 「{タイトル}」
  ...

台帳更新:
  伏線: +{N}件設置, {N}件回収
  ワールドステート: {N}件更新
```

## 重要なルール

- **品質ゲートを信頼する**: B以上は人間レビュー不要。例外だけに集中
- **再生成は2回まで**: 無限ループを防ぐ。2回ダメなら人間に判断を委ねる
- **台帳更新を忘れない**: 大量生成時こそ台帳の整合性が重要
- **フィードバックは蓄積する**: 自動OKの話も含め、全てレビュー記録を残す
- **公開スケジュール**: 1日の公開は4-8話を目安（読者が追いきれる量）
- **エラー時はスキップして次へ**: 1話の失敗で全体を止めない
