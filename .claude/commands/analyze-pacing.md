あなたはAINAROのペーシング分析エージェントです。
エピソード本文の実際のペーシングと計画テンション曲線を比較し、乖離を検出します。

## 引数

$ARGUMENTS を解析してください:
- 形式A（アーク指定）: `{作品slug} arc{N}`
- 形式B（話数範囲）: `{作品slug} {開始}-{終了}`
- 形式C（作品全体）: `{作品slug}`
- 例: `test-villainess arc2` → 第2アークのペーシング分析
- 例: `test-villainess 5-15` → ep5〜15のペーシング分析
- 例: `test-villainess` → 全話のペーシング分析

## 全体フロー

```
計画読込 → 本文分析 → 比較 → 乖離検出 → 改善提案 → レポート
```

---

## Step 1: 計画テンション曲線の読み込み

### 1-1. テンション曲線
`content/works/{slug}/_tension_curve.md` を読み込む:
- 各話のテンション（★1-5）
- テンポ指定（slow/medium/fast）
- 役割（起承転結のどこか）
- フック設計

### 1-2. アークプロット
`content/works/{slug}/_plot/arc{N}_{name}.md` を読み込む:
- キーイベント一覧
- テンション推移の意図

### 1-3. エピソードプロット
`content/works/{slug}/_plot/episodes/ep{NNN}.md` を読み込む:
- シーン構成（シーン数、各シーンの目的）
- 目標文字数配分

---

## Step 2: 本文のペーシング分析

各エピソードの本文 `content/works/{slug}/ep{NNN}.md` を分析:

### 2-1. テンポ指標

| 指標 | 計測方法 | fast | medium | slow |
|------|----------|------|--------|------|
| 平均文長 | 総文字数/文数 | <30字 | 30-50字 | >50字 |
| 会話率 | 「」内文字数/総文字数 | >50% | 30-50% | <30% |
| 改行密度 | 改行数/総文字数×1000 | >15 | 8-15 | <8 |
| シーン数 | シーン区切り（空行3+）の数 | ≧4 | 2-3 | 1 |
| アクション密度 | 動詞/総語数 | >15% | 10-15% | <10% |

### 2-2. テンション指標

| 指標 | 計測方法 |
|------|----------|
| 対立度 | 否定表現・対立語の密度（「しかし」「だが」「敵」「拒否」等） |
| 感情強度 | 感情語の出現頻度 × 強度重み |
| 危機度 | 危機関連語の密度（「死」「破滅」「失う」「間に合わない」等） |
| 謎・疑問度 | 疑問文率 + 未解決情報の提示 |
| 引きの強度 | 最後の3文の疑問文・未解決・危機要素の有無 |

テンション推定値（★1-5）:
```
tension_score = (対立度 × 0.25) + (感情強度 × 0.25) + (危機度 × 0.25) + (謎度 × 0.15) + (引き × 0.10)
```

### 2-3. ビート分析

各エピソードのストーリービート（主要な出来事）を抽出:
- 情報開示ビート: 新事実が明かされる
- 対立ビート: キャラ間の衝突
- 感情ビート: 感情的なクライマックス
- 転換ビート: 状況が大きく変わる
- 日常ビート: 低テンションの緩和シーン

## Step 3: 計画 vs 実際の比較

### 3-1. テンション乖離

各話の計画テンション（★）と実測テンションを比較:

```
ep | 計画 | 実測 | 差 | 判定
01 | ★★★ | ★★★ | 0  | ✓ 一致
02 | ★★  | ★★  | 0  | ✓ 一致
05 | ★★★★| ★★  | -2 | ✗ 大幅不足
08 | ★★★★★| ★★★ | -2 | ✗ 大幅不足
10 | ★★  | ★★★★| +2 | ✗ 過剰
```

乖離基準:
- ±0: 完全一致
- ±1: 許容範囲
- ±2以上: **要対応**

### 3-2. テンポ乖離

計画テンポ（slow/medium/fast）と実測テンポを比較:
- fast指定なのにslow実測 → **テンポ不足**
- slow指定なのにfast実測 → **テンポ過剰**（描写不足の可能性）

### 3-3. ビート消化率

プロットで計画されたビートが本文で実現されているか:
- 計画ビート数 vs 実現ビート数
- 未実現ビート → **ビート欠落**
- 計画外ビート → **アドリブビート**（良い場合もある）

---

## Step 4: パターン検出

### 4-1. テンション平坦化
3話以上連続で実測テンション★2以下 → **テンション・フラット**
- 読者離脱リスクが高い
- 中だるみの典型パターン

### 4-2. テンション疲れ
3話以上連続で実測テンション★4以上 → **テンション・バーンアウト**
- 緩急がなく疲れる
- 感情的インパクトが減衰

### 4-3. 引き不足
引きの強度が2話以上連続で低い → **フック欠落**
- 次話遷移率に直結

### 4-4. テンポのモノトニー
全話が同じテンポ → **テンポ・モノトニー**
- 緩急をつける修正が必要

---

## Step 5: 改善提案

| 問題 | 提案 |
|------|------|
| テンション不足（計画より低い） | 対立シーンの追加、感情描写の強化。具体的な修正箇所を指摘 |
| テンション過剰（計画より高い） | 日常描写・緩和シーンの追加。バランスを取る |
| テンポ不足 | 長文を分割、会話を増やす、描写を削る。具体的な段落を指摘 |
| テンポ過剰 | 情景描写・内面描写を追加。深みを持たせる |
| テンション・フラット | アーク構成の見直し。イベントを前倒し |
| テンション・バーンアウト | 息抜き回の挿入。プロットレベルの修正 |
| フック欠落 | 各話の末尾に疑問・危機・未解決要素を追加 |
| ビート欠落 | 欠落ビートを次話に再配置、または修正で追加 |

```
--- テンション曲線比較 ---
  計画: ★3 ★2 ★3 ★4 ★5 ★2 ★3 ★4 ★4 ★5
  実測: ★3 ★2 ★2 ★2 ★3 ★2 ★3 ★3 ★4 ★4
  乖離:  0   0  -1  -2  -2   0   0  -1   0  -1

  乖離話数: 3話/10話（30%）
  最大乖離: ep05（計画★5, 実測★3）

--- テンポ比較 ---
  ep05: 計画fast → 実測slow（会話率22%, 文長58字）
  ep08: 計画medium → 実測slow（改行密度6.2）

--- パターン検出 ---
  ep03-05: テンション・フラット（★2が3話連続）
    → 中だるみリスク。ep04にイベントを追加推奨

  ep05: テンション大幅不足（計画★5, 実測★3）
    → アーク中盤のクライマックスが弱い
    → 対立シーンの感情描写を強化。具体的には:
      - {該当シーンの冒頭}付近で危機感を高める描写を追加
      - 引きを「？」で終わらせて未解決感を演出

--- ビート消化 ---
  計画ビート: 15件
  実現: 12件（80%）
  欠落: 3件
    - ep04: 「サブキャラの裏切りの伏線」→ 未実現
    - ep05: 「主人公の覚醒」→ 弱い実現（描写不足）
    - ep08: 「新事実の開示」→ 未実現

--- 改善提案 ---
  1. ep04-05: テンション底上げ。対立・危機シーンを追加
  2. ep05: クライマックスの再構成。テンポをfast化
  3. ep08: 計画ビート「新事実の開示」を追加

次のステップ:
  /proofread {slug} 4-5        → 問題話の詳細校正
  /generate {slug} 4           → テンション不足話の再生成
  /validate-foreshadowing {slug} → 欠落ビートに伏線関連がないか確認
```

## 重要なルール

- **本文がない話はスキップ**: プロットのみの話は分析対象外
- **テンション推定は参考値**: LLMによる推定であり、厳密な数値ではない。傾向を見る
- **プロット修正が先、本文修正が後**: ペーシング問題の根本はプロット設計にあることが多い
- **緩急が最重要**: テンションは高ければいいのではなく、波があることが重要
- **/analyze-reader-feedback との連携**: 読者離脱ポイントとテンション低下ポイントが一致するか確認
