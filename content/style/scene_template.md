# シーンプロット テンプレート

> 各エピソードのプロットファイル内で、シーンごとにこの構造を使う。
> シーン単位生成時、エージェントは1シーンずつこの情報を参照して生成する。

## シーン定義フォーマット

```markdown
### シーン{N}: {シーン名}（{場所}）

- 場所: {具体的な場所名}
- 時間: {作中時間帯}
- 登場: {キャラ名のカンマ区切り}
- 目的: {このシーンが果たす物語上の役割。1-2文}
- 文字数目安: {800-2000字の範囲で指定}

#### 感情の弧
{開始の感情} → {中盤の変化} → {終了の感情}

#### 五感描写の指定
- 必須: {最低1つの五感を指定。例: 視覚「月明かりの銀髪」}
- 推奨: {あれば追加の五感。例: 嗅覚「ラベンダーの枕」}

#### このシーンで達成すること
1. {達成項目1}
2. {達成項目2}

#### このシーンで設置/回収する伏線
- 設置: F{ID} {内容}
- 回収: F{ID} {内容}

#### このシーンのアイロニー
- DI{ID}: {読者は知っているがキャラは知らない情報の活用方法}

#### トランジション（次シーンへの繋ぎ）
{このシーンの末尾がどのように次のシーンに繋がるか}
```

## シーン単位生成フロー

### 1話を構成するシーン数

- 通常: 2-3シーン
- クライマックス回: 3-4シーン（ただし1シーンが短い）
- 日常回: 2シーン（ゆったり）

### 文字数の配分例（4,000字の話）

| パターン | シーン1 | シーン2 | シーン3 |
|----------|---------|---------|---------|
| 2シーン構成 | 2,200字 | 1,800字 | - |
| 3シーン構成 | 1,500字 | 1,500字 | 1,000字 |
| 重点シーンあり | 2,500字 | 1,000字 | 500字 |

### 生成順序

```
1. シーン1を生成
   → 即時チェック（口調・テンポ・文字数）
   → 問題あれば修正

2. シーン1末尾 + トランジション指示 → シーン2を生成
   → 即時チェック
   → 問題あれば修正

3. （3シーン構成なら）シーン2末尾 → シーン3を生成

4. 全シーンを結合
   → 全体チェック（テンポの流れ・文字数合計・引きの強度）
   → 必要なら部分リライト
```

### シーン間トランジションのパターン

1. **場所移動**: 「廊下を抜けて、中庭に出た」（物理的移動）
2. **時間経過**: 空行 + 時間を示す描写（「気がつけば、陽が傾いていた」）
3. **視点切り替え**: 同じ場所で別の出来事に注目が移る
4. **カットバック**: シーンを断ち切って次へ（高テンション時に有効）

## 大規模生成時のシーン設計ルール

月5,000話以上の生成では、プロットも自動生成する。
シーンプロットの自動生成時は以下のルールに従う:

1. **シーンの目的は1つ**: 1シーンで達成することは1-2個まで。欲張らない
2. **感情の弧は必須**: 開始→変化→終了の3点を必ず指定
3. **伏線は台帳と連動**: 新規伏線はIDを採番して台帳に追加
4. **五感描写は最低1つ**: 自動生成時も指定を忘れない
5. **文字数を明示**: 暗黙の「4,000字で」ではなく、シーンごとに配分を明記
