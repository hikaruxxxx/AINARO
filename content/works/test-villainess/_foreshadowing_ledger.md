# 伏線台帳：断罪前夜に目覚めた悪役令嬢

> 生成エージェントは毎話この台帳を参照し、未回収伏線を意識して執筆する。
> 回収したらstatusを `回収済` に変更し、回収話数を記録する。

## 凡例

| 列 | 説明 |
|----|------|
| ID | 伏線の一意ID（F+連番） |
| 伏線内容 | 何を仕込んだか |
| 設置話 | 仕込んだ話数 |
| 回収予定 | _plot で想定している回収話数 |
| 回収話 | 実際に回収した話数 |
| status | 未回収 / 部分回収 / 回収済 / 放棄 |
| 重要度 | S（メインプロット直結）/ A（サブプロット）/ B（雰囲気・世界観） |
| tags | セマンティック検索用キーワード（カンマ区切り） |
| メモ | 回収方法のヒントや制約 |

## 運用ルール

1. **生成時**: 未回収伏線のうち回収予定が近いものを確認し、自然に織り込む
2. **新規設置**: 生成後にこの台帳に追記する（IDは連番）
3. **回収判定**: 伏線の核心が明かされたら `回収済`。一部のみなら `部分回収`
4. **放棄判定**: プロット変更で不要になった伏線は `放棄` + 理由をメモに記載
5. **棚卸し**: 10話ごとに未回収伏線を棚卸し。回収予定を過ぎたものは要対応
