## 4. 品質層

### 4.1 スキル: /proofread

生成済��エピソードを一括校正。

```

```

**処理フロー**:
1. ep{num}.md を読み込み
2. 校���エージェント実行（NG表現 + 文体一貫性 + 設定整合性）
3. AI検出エージェント実行

5. 結果をターミナルに表示 + data/feedback/proofread/{slug}_ep{num}.json に保存

**出力フォーマット**:
```

  設定: リゼットの口調パターン一致度 85%

[AI検出] 42/100（低リスク）
[人気予測] B (68点)
  vs 成功作品: 疑問率が低め（3% vs 平均8%）、感嘆率は良好

修正提案:
  1. 会話シーンを1つ追加（現在のシーン2に差し込み推奨）
  2. 疑問文を2-3文追加して読者の好奇心を刺激
```

## 7. フィードバックループ

### 7.1 ループの流れ

```
生成 → 校正(自動) → レビュー(人間) → 配信 → 計測(自動)
  ▲                                              │
  └──────────── ナレッジ更新 ◄────────────────────┘
```

### 7.2 学習の3段階

**即時学習（同日）**: 校正・レビューの修正内容
- /review で記録した修正理由が、次の /generate に反映
- 「テンポが遅い」→ ��回生成で短文率・テンポパラメータを調整

**短期学習（週次）**: 読者行動データ
- 読了率が低い話 → 何が悪かったかを特徴量で分析
- 読了率が高い話 → 何が効いたかを特徴量で分析
- data/feedback/reader_signals/ を集計してベンチマーク更新

**長期学習（月次）**: ベンチマーク再キャリブレーション
- クロールデータの追加（新作の人気作品）
- 自社作品の蓄積データによるベンチマーク補正
- 人気予測モデルの重み調整

### 7.3 自社作品データの特別扱い

自社作品は**すべての指標が揃う**唯一のデータソース:
- テキスト特徴量（生成時に自動計算）
- 校正スコア（校正時に自動保存）
- 人間レビュー（/review で記録）
- 読者行動（配信後に自動収集）

こ���により「テキスト特徴量 → 読者行動」の直接的な因果関係を学習できる。
クロールデータ（なろう等）は「テキスト → ポイント」の粗い相関しか取れないが、
自社デ��タは「テキスト → 読了率・次話遷移率」の精密な相関が取れる。

### 7.4 フィードバックの反映方法

```
[人気予測エージェント]
  初期: クロールデータのベンチマークのみ
  3ヶ月後: 自社��ータの���者行動で重みを再学習
  → 「疑問率が高い話は次話遷移率も高い」等の自社固有の知見

[校正エージェント]
  初期: ハードコード閾値
  1ヶ月後: 修正��歴からNG表現リストを自動更新
  → 「この表現は毎回修正され��」→ blacklist.mdに��動追記

[生成スキル]
  初期: 設定ファイル + 汎用プロンプト
  2週間後: 過去のレビュー/修��を参照して生成
  → 「前回テンポが遅いと言われた」→ 短文率を意識的に上げる
```

## まとめ
