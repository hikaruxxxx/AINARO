# 小説生成アーキテクチャ v2: 月5,000-10,000話規模

## 概要

v1（月100話想定）から v2（月5,000-10,000話想定）への移行。
根本的な変更: **人間がプロットを書く → AIがプロットも自動生成する**。

## スケール計算

| 指標 | v1 | v2 |
|------|-----|-----|
| 月間話数 | ~100話 | 5,000-10,000話 |
| 日間話数 | ~3話 | 170-330話 |
| 同時並行作品数 | 1-3 | 50-100+ |
| 1作品あたり | 30話 | 30-100話 |
| 人間レビュー | 全話 | 例外のみ（5-10%） |
| プロット | 手書き | 自動生成 |

## コマンド体系

| コマンド | 用途 | 規模 |
|----------|------|------|
| `/seed {slug} "{ジャンル}" "{ログライン}"` | ストーリーシード → 全設定自動生成 | 1作品 |
| `/generate {slug} [話数]` | エピソード生成（シーン単位） | 1話 |
| `/proofread {slug} {話数}` | 校正（5観点） | 1-N話 |
| `/review {slug} {話数} {判定} "{コメント}"` | レビュー記録 | 1話 |
| `/daily {slug\|all} [話数]` | 日次運用（生成→校正→レビュー） | 1-全作品 |
| `/batch {new\|existing\|mix} ...` | 大量生成パイプライン | 全作品 |

## コンテキスト窓の管理戦略

月5,000話規模では、1作品の全情報をコンテキストに入れることは不可能。
**階層的な読み込み**で必要最小限のコンテキストを構築する。

### 読み込み優先度

| 優先度 | ファイル | サイズ目安 | 必ず読む |
|--------|----------|-----------|---------|
| 1 | base_guidelines.md | 1KB | ✓ |
| 1 | blacklist.md | 1KB | ✓ |
| 2 | _settings.md | 3KB | ✓ |
| 2 | _style.md | 2KB | ✓ |
| 3 | 該当キャラの該当Phase | 1KB/キャラ | ✓ |
| 3 | 最寄りのワールドステート | 2KB | ✓（あれば） |
| 4 | 該当話のプロット | 1KB | ✓（あれば） |
| 4 | 前話末尾500字 | 0.5KB | ✓ |
| 5 | 伏線台帳（未回収のみ） | 1KB | ✓ |
| 5 | 読者既知情報（直前のみ） | 1KB | ○ |
| 5 | テンションカーブ（該当話） | 0.2KB | ○ |
| 5 | ドラマティック・アイロニー | 1KB | ○ |
| 6 | フィードバック | 0.5KB | △ |

**合計: 約15KB**（全て読んでも。実際はさらに少ない）

### ワールドステートスナップショットの効果

50話生成済みの作品で ep051 を生成する場合:

**v1（スナップショットなし）**: 前話500字のみ。過去49話の情報が消失
**v2（スナップショットあり）**: ep050_snapshot.md（2KB）で50話分の世界状態を圧縮参照

## 品質管理パイプライン

### 自動通過率の目標

| フェーズ | 目標 | 意味 |
|----------|------|------|
| 初期（1-2ヶ月目） | 60% | 設定を調整しながら品質を上げる |
| 安定期（3-6ヶ月目） | 80% | 大半が自動通過 |
| 成熟期（6ヶ月以降） | 90%+ | 人間は戦略判断に集中 |

### 品質が下がったときの対応

1. **NG表現が増加**: blacklist.md に新パターンを追加
2. **口調ブレが増加**: _characters/ のPhase定義を精緻化
3. **テンポ崩れ**: _style.md のパラメータ調整 + 参考例文追加
4. **プロットの単調化**: ジャンル分析 → 新しいパターンを references/ に追加
5. **伏線の矛盾**: 台帳の棚卸し → 矛盾伏線の放棄 or 修正

## スケーリングのロードマップ

### Phase 0.5: 仕組みの検証（現在〜1ヶ月目）

- test-villainess で新構造（v2）の全機能を検証
- /seed で2-3作品を新規作成し、/daily で運用テスト
- 品質ゲートの閾値を調整
- 目標: 月100-300話

### Phase 1: 初期スケール（2-3ヶ月目）

- 10-20作品を並行運用
- /batch で新規作品を週5本ペース
- 自動通過率60%を確認
- 目標: 月1,000-3,000話

### Phase 2: 本格スケール（4-6ヶ月目）

- 50-100作品を並行運用
- 品質フィードバックループの自動化（読者データ連携）
- 目標: 月5,000-10,000話

### Phase 3: 最適化（6ヶ月以降）

- 読者データに基づく自動的な作品優先度調整
- 人気作品に自動でリソース集中
- 不人気作品の早期打ち切り判断
- 新ジャンルの自動探索

## v3 拡張: 自律オーケストレーション + 動的コンテキスト（2026-04-06追加）

v2のコマンド体系を維持しつつ、以下の3レイヤーを追加。

### 拡張1: 自律オーケストレータ（_pipeline_state.md）

各作品に `_pipeline_state.md` を追加。エピソードごとの状態を管理し、`/daily` がステートマシンとして自動的に次アクションを決定する。

**状態遷移:**
```
planned → plot_ready → generating → drafted → proofreading
  → approved → published
  → auto_fixing → (approved | exception_review)
  → regen(max 2) → (drafted | exception_review)
```

**優先度ルール:**
1. exception_review > 3件の作品 → 新規生成停止
2. publish_queue が空の作品 → 最優先で生成
3. 中途状態のエピソードを先に完了
4. 読者指標が高い作品を優先

### 拡張2: 動的コンテキストプロファイル

話の性質に応じて、コンテキストバジェット配分を自動最適化。

| プロファイル | 判定条件 | L3の重点 |
|---|---|---|
| `climax` | テンション★5、決戦・告白 | キャラ状態+伏線を拡大 |
| `daily_life` | テンション★1-2、日常 | 世界観描写を拡大 |
| `revelation` | 真相判明、伏線回収3件+ | 伏線+読者既知+アイロニーを最大化 |
| `arc_transition` | アーク転換点 | ワールドステート+キャラ+タイムライン拡大 |
| `balanced` | デフォルト | 均等配分 |

### 拡張3: 追加ファイル

| ファイル | 用途 | 更新タイミング |
|---|---|---|
| `_pipeline_state.md` | エピソード状態管理 | 毎アクション後 |
| `_timeline.md` | 作中時間軸 | 毎話生成後 |
| `_world_state/ep{N}_delta.md` | 前話からの差分コンテキスト | 毎話生成後 |
| 伏線台帳の `tags` フィールド | セマンティック検索用キーワード | 伏線設置時 |

### v2→v3 移行

1. 既存作品に `_pipeline_state.md` を追加（既存epは `published` として初期化）
2. 既存作品に `_timeline.md` を追加（既存epの時系列を復元）
3. 既存伏線に `tags` を後付けで付与
4. `/daily` のステートマシンモードが自動的に有効化される
