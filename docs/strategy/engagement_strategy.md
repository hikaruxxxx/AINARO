# エンゲージメント・マネタイズ戦略

**「面白さが実証されるまで課金しない」**

---

## 原則

1. **面白くないコンテンツに課金モデルを載せても意味がない**
2. 順番は「面白い → 読者が集まる → 課金が成立する」。逆はない
3. 全機能は「NULLならOFF」設計。スイッチ一つでON/OFFできる
4. 日本のWeb小説読者は「無料」に慣れている。課金の導入は慎重に

---

## フェーズ別運用

### Phase 0: 全話無料 + 面白さ計測（0〜12ヶ月）

#### やること
- **全エピソードの `unlock_at` = NULL、`unlock_price` = 0**（完全無料）
- ポイントシステムはOFF（ログインボーナスもOFF）
- コンテンツ選別ファネルで **3〜5作品 × 各3話のパイロット版** を投入
- 読者行動データを蓄積し、面白さ先行指標を計測

#### 計測する指標
| 指標 | 計測方法 | 判断基準 |
|------|----------|----------|
| 読了率 | complete / start | **60%以上**で合格 |
| 次話遷移率 | next / complete | **40%以上**で合格 |
| シリーズ完走率 | 最新話到達 / 第1話start | 参考値（3話では短い） |
| 平均滞在時間 | completeイベントのreading_time_sec | ジャンル平均と比較 |
| ブックマーク率 | bookmark / unique_users | 5%以上で良好 |

#### コンテンツ選別フロー
```
プロット作成（content_candidates: phase='plot'）
  ↓
パイロット版3話を作品として公開（phase='pilot'）
  ↓
2〜4週間の計測期間
  ↓
スコア自動算出（/api/admin/content-funnel/[id]/evaluate）
  ↓
判定:
  読了率60%+ かつ 次話遷移率40%+ → 連載化（phase='serial'）
  読了率40-60% → 改稿して再テスト
  読了率40%未満 → アーカイブ（phase='archived'）
```

#### A/Bテストの活用
- パイロット版で読了率が基準に届かない場合、**展開Bを用意してA/Bテスト**
- テストはエピソード単位。同じ作品の第2話を2パターン用意し、次話遷移率で比較
- 最低100セッション/バリアントでテスト判定（統計的有意性は簡易判定）

### Phase 1a: ポイント経済圏テスト（12〜15ヶ月）

#### 前提条件
- **読了率60%+、次話遷移率40%+の作品が最低1本存在すること**
- MAU 3,000以上

#### やること
- ログインボーナス（毎日+1pt）を有効化
- エピソード読了ボーナス（+1pt）を有効化
- **課金はまだしない。ポイントは無料で配布するのみ**
- 目的: ログイン率・再訪率の変化を計測

#### 計測する指標
| 指標 | 計測方法 | 成功基準 |
|------|----------|----------|
| DAU/MAU比 | ログインボーナス受取率 | 15%以上 |
| 再訪率 | 翌日以降の復帰率 | 30%以上 |
| ポイント蓄積行動 | pt獲得イベント数/ユーザー | 週3回以上のログイン |

### Phase 1b: 時限解放テスト（15〜18ヶ月）

#### 前提条件
- Phase 1aでDAU/MAU比15%以上を達成
- 連載中の人気作品（読了率60%+）が2本以上

#### やること
- 人気作品の**新規エピソード（第N+1話以降）**に時限解放を設定
  - `unlock_at` = 公開から24時間後
  - `unlock_price` = 3ポイント
- 既存エピソード（第1話〜第N話）はそのまま無料
- **最初の3話は常に無料（フック）**

#### 計測する指標
| 指標 | 計測方法 | 成功基準 |
|------|----------|----------|
| ポイント解放率 | unlock数 / ロック表示数 | 5%以上 |
| 待機後読了率 | 24h後のstart率 | 70%以上（待ってでも読む） |
| 離脱率変化 | ロック導入前後のdrop_rate比較 | 10%以内の増加 |
| 読者の継続率 | MAU変化 | 5%以内の減少 |

**判断基準**: ポイント解放率5%以上 かつ MAU減少5%以内なら継続。MAU10%以上減少なら即時ロールバック。

### Phase 1c: ポイント購入の導入（18ヶ月〜）

#### 前提条件
- Phase 1bで離脱率が許容範囲内
- ポイント解放の利用者が一定数存在

#### やること
- Stripe連携でポイント購入を開始
  - 100pt = 500円
  - 300pt = 1,200円（20%お得）
  - 500pt = 1,800円（28%お得）
- サブスク（月額500円、毎月100pt付与 + 広告非表示）

## ポイント経済の設計

### 獲得手段（全て無料）
| アクション | 獲得pt | 制限 |
|------------|--------|------|
| ログインボーナス | +1 | 1日1回 |
| エピソード読了 | +1 | 同一エピソードは1回のみ |
| コメント投稿 | +1 | 1日1回 |

### 消費
| アクション | 消費pt | 備考 |
|------------|--------|------|
| エピソード先読み解放 | 1〜5 | エピソードごとに設定可 |

### バランス設計
- 毎日ログイン+1話読了で **1日2pt** 獲得
- 先読み1話 = 3pt → **毎日ログインすれば2日に1話は先読みできる**
- 週7話更新の作品で、無料ユーザーは週3〜4話を先読み可能
- 残りの3〜4話は24h待てば無料で読める
- 「もっと早く読みたい」人だけがポイント購入する

## 競合ベンチマーク

### 日本市場
| プラットフォーム | 広告 | 先読み課金 | 作者還元 |
|---|---|---|---|
| なろう | バナー | なし | 0% |
| カクヨム | 控えめ | なし | PV分配（微小） |
| アルファポリス | あり | アプリのみ | インセンティブ+印税8-10% |

### 海外市場（参考）
| プラットフォーム | 1話単価 | Wait-to-Read | 作者還元 |
|---|---|---|---|
| Tapas | $0.30-0.80 | 72h | 50-60% |
| Radish | $0.60-1.50 | 一部あり | 約50% |
| WebNovel | $0.10-0.40 | なし | 30-50% |

### Novelisのポジション
- **日本で「先読み課金」を小説に適用した成功例はまだない**
- 漫画アプリ（ピッコマ、LINEマンガ）ではWait-to-Readモデルが年間数百億円規模で成立
- **漫画で実証されたモデルを小説に適用する**のがNovelisの賭け
- 90%還元は日本・海外とも最高水準。ただし読者基盤がなければ意味がない

## リスクと撤退基準

### 課金テスト（Phase 1b）の撤退基準
- MAU 10%以上減少 → **即時ロールバック**（全話無料に戻す）
- 読了率が全作品で10%以上低下 → ロールバック
- 3ヶ月テストして解放率1%未満 → モデル再設計

### 最悪シナリオ: 課金が成立しない場合
- 広告収入のみで運営（Phase 0の計画に戻る）
- 自社制作コンテンツの利益率100%なので、広告だけでも黒字化は可能
- ポイントシステムは「ログインボーナスによる再訪促進」としてのみ運用
