# プロット骨格テンプレート(共通仕様)

## 目的

各サブジャンルの `{subgenre_id}.md` は、Layer 2(プロット骨格)生成時に LLM へ渡す **構造ガイド**。
Web小説特化プロット構造(第1アーク完結 + 全体引き)に従う。

## 必須セクション

各テンプレファイルは以下のセクションを必ず持つ:

```markdown
# {サブジャンル名}

## 起点(opening)
主人公の境遇と読者の感情移入フックをどう設計するか。
- 例示2つ
- 避けるべきパターン

## 転換点1(inciting incident)
ログライン要素の発動。物語の起点となるイベント。
- いつ発動させるか(ep1のどこか / アーク全体のどこか)
- 例示2つ

## 転換点2(commitment)
主人公の決意と方向性の確定。読者の期待を固める段階。
- 例示2つ

## 第1アーク完結(arc climax)
小目標達成とカタルシス。アーク内で完結させる要素。
- 例示2つ
- 完結させすぎない注意点

## 全体引き(hook to next)
大目標への布石。続きを読みたくなる要素。
- 例示2つ

## ジャンル固有の禁じ手
- 該当ジャンルでよく失敗するパターン
- batch_002 で観測された問題があれば記載

## 評価軸との対応
- どの評価軸を満たすために何を書くべきか
- eval-weights-by-genre.json の特化軸との対応
```

## 運用ルール

- 1ファイル500〜800字程度を目安(LLMに渡しても context を圧迫しない)
- 例示は具体的に(抽象論より「処刑された王女が3周目のループで真犯人を暴く」のような)
- 改善は本ファイルではなく各サブジャンルファイルで行う
- batch_NNNNNNNN_NNN の結果を見て週次レビューで更新
